# ユーザーニーズ洗い出し

## 機能的ニーズ（何ができるべきか）
- コードを安全にアップロード/貼り付けし、即時にAIレビューを受け取れる
- 指摘内容がカテゴリ（バグ・設計・パフォーマンス・アクセシビリティ）で整理される
- 修正例やリファレンスリンクが提示される
- レビュー履歴が保存され、比較・再レビューができる
- サインイン後もセッションが維持され、学習時間を節約できる
- レビュー結果をメールや共有リンクで送れる

## 情緒的ニーズ（どう感じたいか）
- 「時間を無駄にしていない」と感じられる即時性
- 「自分でも改善できる」と思える具体性と優先度の提示
- 「評価されている」という安心感（セキュアな認証とデータ保持）
- 「学びが残る」という再現性（履歴・メモ・再レビューのしやすさ）
- 過剰な専門用語を避けた、読みやすいフィードバックトーン
