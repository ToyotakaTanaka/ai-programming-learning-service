| ナビゲーション | モバイルではハンバーガーメニュー | `AppBar`, `Toolbar`, `IconButton` |

### 2. ヒーローセクション

| 要素 | 説明 | MUIコンポーネント |
|------|------|-------------------|
| キャッチコピー | インパクトのある主訴求 | `Typography variant="h1"` |
| サブタイトル | 価値提案の補足説明 | `Typography variant="h5"` |
| Primary CTA | 新規登録への誘導 | `Button variant="contained" size="large"` |
| Secondary CTA | 詳細情報への誘導 | `Button variant="outlined" size="large"` |
| 背景 | グラデーション or 抽象パターン | `Box` with CSS gradient |

### 3. 特徴セクション

| 要素 | 説明 | MUIコンポーネント |
|------|------|-------------------|
| セクションタイトル | 「CodeMentor AIの特徴」 | `Typography variant="h2"` |
| 特徴カード×3 | アイコン+タイトル+説明 | `Card`, `CardContent`, `Avatar` |
| グリッドレイアウト | レスポンシブ配置 | `Grid2` (3列→1列) |

**3つの特徴（ペルソナ・ニーズ対応）:**
1. **即時AIレビュー** - コードを貼り付けるだけで数秒でフィードバック（ペルソナ「時間が足りない」課題に対応）
2. **学習履歴の蓄積** - ポートフォリオ作成・インターン応募前の振り返りに最適（ペルソナBの動機に対応）
3. **カテゴリ別の改善提案** - バグ・設計・パフォーマンス・アクセシビリティで整理（機能的ニーズ「カテゴリ整理」に対応）

### 4. 使い方ステップセクション

| 要素 | 説明 | MUIコンポーネント |
|------|------|-------------------|
| セクションタイトル | 「使い方は簡単4ステップ」 | `Typography variant="h2"` |
| ステッパー | 視覚的な順序表現 | `Stepper`, `Step`, `StepLabel` |
| 説明テキスト | 各ステップの詳細 | `Typography` |

**ステップ構成（ユーザージャーニーマップ対応）:**

3. AIレビュー結果を確認（問題点・改善例・参考リンク）
4. 修正して再レビュー（履歴比較で成長を実感）

### タイポグラフィ

| 要素 | フォント | サイズ (Desktop / Mobile) |
|------|----------|---------------------------|
| H1 (Hero) | Inter Bold | 48px / 32px |
| H2 (Section) | Inter SemiBold | 36px / 28px |
| H3 (Card Title) | Inter Medium | 24px / 20px |
| Body | Inter Regular | 16px / 14px |
| Caption | Inter Regular | 14px / 12px |

### レイアウト原則

1. **8pxグリッドシステム** - すべての余白・サイズは8の倍数
2. **最大コンテンツ幅** - `maxWidth="lg"` (1200px) で可読性維持
3. **セクション間余白** - 80px (Desktop) / 48px (Mobile)
4. **カード間余白** - 24px (Desktop) / 16px (Mobile)

## ダークモード対応方針

### 実装方法

```typescript
// MUIのuseThemeフックで現在のモードを取得
const theme = useTheme();
const isDarkMode = theme.palette.mode === 'dark';

// 条件付きスタイリング
<Box
  sx={{
    bgcolor: 'background.default',
    color: 'text.primary',
  }}
>
```

### 対応ポイント

1. **システム設定の自動検出** - `prefers-color-scheme` メディアクエリ対応
2. **手動切り替え** - ヘッダーにトグルボタン配置（将来実装）
3. **画像・アイコン** - モード別に最適化されたアセット使用
4. **コントラスト比** - WCAG 2.1 AA基準（4.5:1以上）を維持

### WCAG 2.1準拠項目

| 基準 | 対応方法 |
|------|---------|
| 色のコントラスト | 4.5:1以上を確保 |
| キーボード操作 | すべてのインタラクティブ要素にフォーカス可能 |
| スクリーンリーダー | 適切なaria-label、alt属性 |
| フォーカス表示 | 視認性の高いフォーカスリング |

## パフォーマンス考慮

1. **アニメーション最小化** - 初回ロード時のフェードインのみ
2. **画像遅延読み込み** - Below the foldの画像は`loading="lazy"`
3. **コンポーネント分割** - 大きなセクションは別ファイルに分離（将来）
4. **Tree Shaking** - MUIの個別インポートで未使用コード排除
